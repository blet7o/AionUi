## 2025-03-18 - [CRITICAL] Authentication Bypass via Length Extension in constantTimeVerify
**Vulnerability:** The `constantTimeVerify` function in `AuthService.ts` used `padEnd` with `'0'` to equalize string lengths before buffer conversion. This allowed a shorter password to match a longer expected password if the longer password ended in zeros. Specifically, `"a".padEnd(2, "0")` evaluates to `"a0"`, which matches `"a0".padEnd(1, "0")` (which is also `"a0"`). This logic flaw permitted authentication bypasses.
**Learning:** Using string manipulation like `padEnd` to equalize lengths for constant-time comparison is inherently unsafe as it fundamentally alters the values being compared, introducing logical collisions (length extension vulnerabilities) and modifying the underlying secrets.
**Prevention:** Convert secrets directly to buffers first. If the buffers are of unequal length, perform a dummy constant-time comparison (`crypto.timingSafeEqual(expectedBuf, expectedBuf)`) to prevent timing leaks, and always return `false`. Only proceed with the actual `crypto.timingSafeEqual(providedBuf, expectedBuf)` if lengths are identical.
